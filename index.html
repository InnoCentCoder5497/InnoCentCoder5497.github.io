Review Classification
Previous Next

The website use state of the art Neural Networks to classify your reviews about restraunts and food. You are welcome to use this site to predict how much stars your review is worth. Using this website is pretty straight forward. All you have to do is write your review in the textarea given above and press Submit button and you will get the stars prediction our system thinks your review is worth.

Technical Details

The system follows the steps given below to classify reviews and get the probability of review to belong to any star rating. When the review is entered and submit is clicked, the following steps occur at the server

Tokenization
Tokenization is the task of chopping it up into pieces, called tokens , perhaps at the same time throwing away certain characters, such as punctuation. The system uses spacy, a free, open-source library for tokenization of the input review.
Once the review is converted to tokens, the tokens are mapped on integer indexes according to the vocabulary in use by the system. At this point the system is ready to feed the review into the model.
Running model
The model is then fed the indexed review. The model is a Bi-directional LSTM neural network. The system uses the output of BiLSTM and outputs the probablity of the review belonging to each class.
Display Result
At last, the system displays the results it has infered from gthe model in the form oa a simple bar chart.

EMS with Biometrics Attendance
Previous Next

Manual handling of employee information poses a number of challenges. The use of paper work in handling the processes could lead to human error and not forgetting the fact that this is time consuming. These problems can be tackled by designing and implementing “Employee Management System with Biometric Attendance”. The project is aimed at setting up employee information system about the attendance and status of the employee in order to help monitor the performance and achievements of the employee through a password protected system. This system will maintain employee information in a database by fully privacy and authority access. Using Biometric attendance system allows employees to clock in and out using their fingerprint. This can be done with the help of Fingerprint Module and Arduino microcontroller. This fingerprint sensor module will make adding fingerprint, detection and verification super simple. Pairing a biometric device with attendance software allows for more efficient attendance processing. Rather than spending hours adding up manually submitted clock-in and clock-out times, a biometric enabled attendance software will allow you to automatically import data into your system. This will significantly cut down on processing time and will eliminate errors derived from manually inputting information.

Details
Semi-Autonomous Car
Previous Next

This system is concerned with a monocular vision semi-autonomous Car prototype using Raspberry Pi as a processing chip. The need for this system arouse because there are many areas around us and in the world where a Human cannot go itself. The Car will be able to move through those areas that cannot be wandered by the humans. In military, hostile areas can be surveillance by this Car as it is very compact in size and provide live camera feed from the hostile areas. By deploying it, it can work very easily through areas avoiding obstacles in its way through a swift control from the user. The complexity in handling control of the Car is reduced by employing it with game like movement controls and providing assistance by using an ultrasonic sensor.

Camera attached to Raspberry pi sub-system which capture the image and provide system. System sends the data in form of the video to network subsystem over to the user. Mainly Raspberry pi sub-system allows the user to see where the car is actually going so that the user can generate commands for the movement of the car. The commands generated by the user are redirected to the raspberry pi sub-system from the Network sub-system. Raspberry pi sub-system is also detecting the obstacle in back of car using ultrasonic sensor and calculates the distance between the obstacle and the car. If sufficient distance is not available to move car backward the command from Raspberry pi is forwarded to motor driver to disallow the backward movement else this command are rejected. This allows the Car to move backward without being hit by any obstacle.
